Troubleshooting

SymptomLikely CauseWhat To Do
ParserError: '<' operator is reservedPasted placeholder like <THREAD_ID> literally.Use the real UUID in quotes.
Cannot validate argument 'db-index-check'PATH resolved global pg CLI, not repo script.Use .\pg.ps1 db-index-check with local prefix.
The term '.\pg.ps1' is not recognizedTerminal is not in repo root directory.cd to project root, then retry.
PG Self Check reports blockersPost-write policy found gating findings.Fix blockers using rule IDs, rerun strict self-check.
SHOW: The term 'SHOW' is not recognizedSQL typed directly into PowerShell.Run SQL in psql/pgAdmin, not shell.
Terminal stuck on '>>'PowerShell multiline continuation mode.Press Ctrl+C, rerun full command on one line.
No connection could be made (127.0.0.1:8787)Backend is not running.cd server && npm run start
invalid bearer tokenMissing/expired token in worker state.Run governance-login again.
No access token availablePG CLI lifecycle state missing/expired..\pg.ps1 login then retry.
Slack thread not foundThread ID or option key is stale.Create fresh /pg thread, copy exact ID.
Operation timed out (3 seconds)Slack didn't receive fast ack.Check /health and tunnel, then retry.
/pg dispatch_failedSlack couldn't process response in time.Check /health + /integrations/slack/health.
/pg not triggering from composerSlash command not first token.Start message with /pg ... exactly.
TS errors stay after compile passesTS language service is stale.Run Narrate: Restart TypeScript + Refresh Trust Score.
PG Push blocks commit messageGeneric or non-conventional message.Use type(scope): summary format.
PG Push blocked by Dead Code GateHigh-confidence unused code detected.Use "Apply Safe Fixes + Recheck" in gate prompt.
No new governance eventsNothing pending (not an error).Create/finalize a thread, rerun worker.
