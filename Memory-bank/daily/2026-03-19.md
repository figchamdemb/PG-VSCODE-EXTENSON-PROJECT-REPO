## Work Summary
- Added customer-facing GBP billing clarification to the public pricing page and secure portal billing panel.
- Kept billing behavior unchanged while making it explicit that international cards can still pay and any local-currency conversion is typically handled by the buyer bank or card provider.
- Re-ran required repo validation and regenerated Memory-bank artifacts after the copy update.
- Added a configurable referral-program layer on top of the existing affiliate system so valid referral codes can discount annual checkout, referrers earn at least a minimum per paid conversion, and higher paid-referral counts unlock stronger commission tiers.
- Cleared the recurring extension startup popup by fixing the specific strict startup blockers and refreshing the repo map-structure artifacts.
- Added local `AGENTS.md` protection using a sealed SHA-256 hash plus read-only file enforcement, verified automatically during startup.
- Fixed a Narrate Trust Score auth mismatch so server-backed trust evaluation now uses the signed-in secret-storage token instead of incorrectly requiring a separate unexposed settings-only token.
- Fixed a startup regression where normal Memory-bank refresh writes made the map-structure gate look stale and repeatedly triggered `PG Start Failed`; startup now auto-refreshes map-structure artifacts before failing strict startup.
- Removed redundant extension manifest activation events so VS Code stops reporting the repeated `package.json` activation warnings.
- Fixed a false-negative verification regression in `pg narrate-check` so the flow checker now accepts VS Code auto-generated command activation instead of requiring removed redundant `onCommand:` manifest entries.
- Extended the frontend integration workflow so `backend-start` and `frontend-start` can optionally enter the existing 30-second local watch loop with `-Persistent`, letting each role keep heartbeating from its own terminal or agent session without a second explicit `integration-watch` command.
- Added clean stop/end lifecycle controls for autonomous integration workers so a second local command can request `stopped` or `completed` exit on the next heartbeat cycle without killing the shell, while keeping the runtime control signal in `Memory-bank/_generated/frontend-integration-runtime.json`.
- Hardened the frontend integration workflow against tampering by encrypting the local runtime control file, redacting authenticated local ledger/export artifacts, and requiring a short-lived server worker lease plus active integration entitlement on authenticated sync.
- Reworked extension startup enforcement so it stays mandatory per workspace until explicitly stopped, survives restart, and re-prompts fail-closed when PG context disappears or breaks.
- Root `AGENTS.md` or `Memory-bank/` evidence now keeps a workspace in local enforcement even when `pg.ps1` is missing, while healthy auto-start still runs through the normal PG context.
- Added explicit extension commands to stop or resume mandatory workspace enforcement locally.
- Added exact-view source<->narration selection sync so matching lines now highlight and auto-scroll together in the Narrate reading experience.
- Added PG-branded terminal stop/resume commands for the same workspace enforcement state through a generated bridge file watched by the extension.
- Renamed the installed extension display title to `PG-Narrate`, bumped the VSIX version to `0.1.1`, and installed the updated package into normal VS Code so operators can verify the latest build visually.
- Clarified shell-specific PG usage so PowerShell users run `.\pg.ps1 ...` while plain CMD users run `pg.cmd ...`, avoiding unnecessary confusion around `.ps1` execution syntax.
