- Clarified that local Prisma usage in this repo means the local Node server process is using Prisma as its store adapter, not that the database itself must be local.
- Confirmed the current trust pipeline is intentionally server-backed for paid/product enforcement, and adjusted the extension UX so missing authentication is shown as sign-in-required instead of a misleading red trust failure.

- Verified live PostgreSQL state after the schema cleanup and documented that the dedicated `narate-enterprise` database with schema `narate_enterprise` is the only supported runtime target.
- Recorded that the legacy `egov.narrate` schema was an empty retired subset and should no longer appear in repo/operator guidance.
- Hardened extension startup UX so the automatic strict startup flow suppresses local-only dev-profile notices and shows sanitized blocker text instead of raw PowerShell noise.
- Hardened Stripe runtime secret handling so real test/live keys can stay in `server/.env` or deployment vaults, while optional admin-board persistence now uses encrypted-at-rest storage behind `STRIPE_RUNTIME_VAULT_KEY` instead of plaintext runtime JSON.

- Fixed the public pricing UX gap where plan cards had no visible buy/contact actions by wiring pricing-page CTA buttons into `/app` billing and support tabs with query-prefilled plan/module or quote-request context.
- Added staged TapSign portal login UX so the auth shell now shows `Sign Up with TapSign` above GitHub/Google and signed-in users get a persistent reminder to finish TapSign device protection after any existing provider login while the SDK is still pending.
- Added a short GBP billing note to the public pricing page and portal billing panel so customers understand international cards can pay while FX conversion may appear through their bank/card provider rather than as app-side currency switching.
